Class Descriptions

Group

A Group represents a collection of Users and other Groups that, when taken together, form a hierarchy.  It also contains information about the Group itself, such as a name, and interfaces with other parts of gus that can provide information for the Group to use.

Group -> WebPageData Association

Each Group has customized data for its web page, which is stored and edited via the WebPageData class.

Group -> DatabaseConnection Association

Each group stores its data in a database, which is accessed through the DatabaseConnection class.

Group -> Calendar Association

Each group has its own calendar, which is represented by the Calendar class.

Group –> MoneyTracker Association

Each MoneyTracker can store fee or dues data, which needs to be accessible on a per-Group basis.

Group—Group Association

Each Group can have sub-Groups and super-Groups (barring the root Group, which can only have sub-Groups).

User
            This class holds a User’s name, email address, phone number, and their activity status. Users can be members of 1 or more groups, vote in Polls created by their group’s officers, view information on groups, and view events on their group’s calendars.

Officer
            A special kind of User that has special permissions within a group. They may also be the group owner.

DatabaseConnection
            A programming interface for the database.  This database is the one which stores all of the group information.

     Database Stores User Information
                Gets the information of Users as they are created and modified and sends it to the
         DatabaseConnection.  There it is stored in the database.

GroupTemplate
            A special kind of group that has its settings and options preset for specific group types.

WebInterface
            This will display the information described in the class WebPageData on the webpage.

     Interface Draws Data From Database
                The WebInterface needs information to display for users.  This is the connection where the
          WebInterface contacts the DatabaseConnection, and retrieves the information required off of the
          database.

Calendar
            This class holds 0 to many of the Event class and can add, edit, or remove events.

Event
            This class holds an event’s title, date, description, and can send messages.

Reminder
            This class holds 0 to many reminders for a specific event. It holds a date to send the reminder, the users to send it to, and the message to send.

WebPageData
            This is the information required to display a webpage.  The HTML code for the page ect.

MoneyTracker
            This is where GroupOfficers/Owners will be able to post if someone has paid their fees or not.  It does not store any financial data, it is simply a placeholder for groups which have fees associated, and need a way to keep track of whether or not each individual member has paid their fees.
