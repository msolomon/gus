Add Wiki to Group

Actors: Group owner, server

Goals: Add a group wiki for collaboration, communication, etc. within the group

Preconditions: Group owner owns a group, group does not yet have a wiki, group owner is logged into gus.

Summary: A wiki can be added to a group for collaboration, documentation, communication, entertainment, or anything else the group wants. Users that are members of the group will be able to edit the wiki freely and other members will be able to view the changes.

Related use cases: Edit wiki, delete wiki

Steps:

  1. Group owner opens the group management menu.
  2. Group owner checks the box to enable a group wiki and adjusts permissions for the wiki.
  3. Group owner saves changes.
  4. Server adds a pre-generated "main" page to the wiki and adds a link to the group page.

Alternative:

  • During group creation, group owner can start at step 2 and continue as normal.
  • At step 3, group owner can choose not to save changes and the process stops.

Fail: If connection times out the user will be informed by their browser and will have to try to add the wiki again. If an error occurs on the server side, the user will receive a message indicating what went wrong and whether they should try again or alert the system administrator.

Edit Wiki

Actors: User, server

Goals: Edit the content of the group wiki

Preconditions: Group has a wiki, user is a member of that group, user has permission to edit the wiki, user is logged in.

Summary: If the user wants to add content to the wiki, they can edit wiki pages or add new pages to the wiki.

Related use cases: Add wiki to group, delete wiki

Steps:

  1. User finds the page of the wiki they want to edit.
  2. User clicks the "edit" button on the page.
  3. User uses the built in editor to adjust the contents of the page.
  4. User saves changes.
  5. Server updates contents of the page to match the user's edits and takes the user back to the edited page.

Alternative:

  • At step 1, user can choose to create a new page in the wiki instead of editing an existing one.
  • At step 4, user can choose to preview changes before they're made to the wiki.
  • At step 4, user can choose to cancel making changes.

Fail: If connection times out the user will be informed by their browser and will have to try to edit the wiki again. If an error occurs on the server side, the user will receive a message indicating what went wrong and whether they should try again or alert the system administrator. If the user doesn't have permission to edit the wiki, the server will let them know and prevent them from editing the wiki.

Update Document

Colby Blair

Actors: Editing user, group web page, other group users

Goals: User makes change document, possibly with many users at one time

Preconditions: User must me logged in, user must meet group criteria for updating document.

Summary: User wants to edit a GUS document, and clicks the document link on the group's document section of their website to view and edit the document

Related use cases:

Steps:

  1. User navigates to document section of group web page
  2. User clicks on document link
  3. Group web page displays document content
  4. User types to make document changes

    • If not, file is locked in the database in case it is shared. Other users can communicate in document chat, and can view document.

Alternative: User deletes document, user locks file but doesn't save changes.

Fail: User opens document, but loses internet connection. Upon reconnection, user can try to save document, or merge if changes have been made in the mean time. If conflicts arise, a diff can be displayed and user can choose which changes to keep.
