Class Descriptions

 

Site Visitor – consists of an IP address, a timestamp, and a location. The Site Visitor’s IP address and time they accessed the site will be logged and their location will determine whether or not they are allowed to join a group (i.e. a Site Visitor in New York would not be able to join the University of Idaho’s groups).

Member – consists of the member’s name, email address, and user role. A Member inherits all of the attributes of a Site Visitor (i.e. the ability to browse group websites, etc). A Member is an approved user for a group. Upon joining a group, a Member will have a default Role that defines (through the Control Module) what permissions that they have. The Group Leader may also change the Member’s Role.  A Member may also manually send a notification to Members of the Group.

Group Leader – consists of the leader’s Name and email address. A Group Leader inherits all of the attributes of a member. When a group is created, the Group Leader is given default admin permissions.

Group – consists of a Name, the Group Leader’s Name, and a list of Members of the group. A Group may consist of zero or more Groups (sub-groups) and also may consist of zero or more Members (though a Group must have a Group Leader). A Group also consists of a Calendar made up of Events specific to the Group.

Calendar – consists of zero or more Events, an addEvent method, and a remove Event method, giving the ability to edit Events in the Calendar. A Group has one Calendar that can be updated with Events for that Group.

Events – consists of an Event Name. Events are listed on the Group’s Calendar. For any given Event, a Notification can be automatically sent to remind Members of the Event.

Notifications – consists of a notification, which is a message that is sent to specified Members of the Group.

Role – consists of a list of access permissions based on the control module. A Role is what privileges/permissions a Member has.

Control Module – consists of a list of allowed class methods. These are the exact methods Members are able to perform via their Role.

Contact List – consists of a list of Members, an addContact method, a deleteContact method, and a displayList method. A Contact List is a list of Members created by the Group Leader specifying different Members to contact. For example, a Contact List may exist for the whole Group or just for the main officers of a group. Contacts may be removed and added, and the list can be published to the Group.

 

Web Interface

Site Front End   - consists of a style and a layout. It is the control class to display front end content.

Layout – consists of an ID, pages, a layout description, and an external layout. It is the control class to manage front end content layout.

Feed Element – consists of a feed url, a feed type, a max feed (5), a style wrapper header, and a style wrapper footer. It is a type of element to display dynamic data from external sources.

Page – consists of a list of elements.

Element – consists of an ID, html, a type, a label, and a determiner for input. It is a building block of html content.

Content Manager – consists of creation methods for layout. It is the control Class for managing front end content.

Form – consists of a list of elements and a url. It is a collection of input elements that can be submitted to a Group.

Post – consists of a timestamp, message content, attachments, and the owner name. It is a message associated with a user/thread.

Thread – consists of a list of posts. It is group of messages associated with a given forum.

Forums – consists of a list of threads. It is an online message board for a given group.

Image Gallery - consists of a list of images and group (acting as the owner) . It is a collection of images associated with a particular group.

Image – consists of an image file, image description, and the image gallery it is in. It is a visual element belonging to an Image Gallery.
